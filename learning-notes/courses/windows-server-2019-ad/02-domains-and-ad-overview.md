## Overview  

Section 2 introduces Windows domains and Active Directory, including domain controller setup and the core structure of AD used for managing users, computers, and resources.

## Module 2.1: What is a Domain and Active Directory (AD)?  

**Learning Objectives:**  
- Understand what a Windows domain is and why Active Directory is used  
- Identify the core components and purpose of Active Directory in enterprise networks  

**Key Topics:**  
- Definition of a Windows domain  
- Purpose of Active Directory Domain Services (AD DS)  
- Centralized identity and authentication  
- Basic AD components (users, computers, domain controllers)  
- How domains enable centralized management and security

## Module 2.2: Building a Domain Controller  

**Learning Objectives:**  
- Promote a Windows Server 2019 instance to a domain controller  
- Create a new Active Directory domain in a lab environment  

**Key Topics:**  
- Installing the Active Directory Domain Services (AD DS) role  
- Creating a new forest and domain  
- Domain and forest functional levels  
- DNS integration during domain controller promotion  
- Initial domain configuration and verification

## Module 2.3: AD Overview  

**Learning Objectives:**  
- Understand the logical structure of Active Directory  
- Identify core AD components used for organizing and managing objects  

**Key Topics:**  
- AD logical structure (domains, trees, forests)  
- Organizational Units (OUs) and object hierarchy  
- Users, groups, and computer objects  
- Basic administrative delegation concepts  
- How AD structure supports scalable management

## KEY TAKEAWAYS

- Active Directory provides centralized identity and access management.  
- Domain controllers host and enforce AD services.  
- AD structure (domains, forests, OUs) enables scalable administration.
