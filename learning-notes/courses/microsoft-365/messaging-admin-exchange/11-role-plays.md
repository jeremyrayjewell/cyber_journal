## Module 11.2: Role Play 1: Meeting with the IT Director on setting up a shared mailbox  
**Learning Objectives:**  
- Explain what a shared mailbox is  
- Understand when a shared mailbox should be used  
- Communicate the business value of shared mailboxes to non-technical stakeholders  

**Key Topics:**  
- Shared mailbox purpose:
  - A mailbox used by multiple users  
  - No separate login required  
  - Accessed through user permissions  
- Common use cases:
  - Team inboxes (info@, support@, payroll@)  
  - Executive assistants managing mail for leaders  
  - Departmental communication  
- Benefits:
  - Centralized communication  
  - Easier collaboration  
  - No additional license required (within size limits)  
  - Simplifies access control and auditing  
- Real-world framing:
  - Improves workflow for directors and assistants  
  - Reduces mailbox duplication  
  - Keeps business communication organized

## Module 11.3: Role Play 2: Meeting with co-administrator on how PIM improves role-based administration  
**Learning Objectives:**  
- Explain what Privileged Identity Management (PIM) is  
- Understand why permanent admin access is risky  
- Describe how PIM improves security for Exchange administrator roles  
- Communicate PIM in simple, real-world terms  

**Key Topics:**  
- Purpose of PIM:
  - Provides *just-in-time* administrative access  
  - Removes permanent admin privileges  
  - Reduces security risk from compromised accounts  
- How PIM works:
  - Users activate roles only when needed  
  - Access is:
    - Time-limited  
    - Audited  
    - Logged  
  - Often requires:
    - MFA  
    - Approval  
    - Justification  
- Exchange admin use case:
  - Exchange Admin role is not always active  
  - Activated only during:
    - Configuration changes  
    - Troubleshooting  
    - Investigations  
- Security benefits:
  - Smaller attack surface  
  - Lower risk of privilege abuse  
  - Clear audit trail of admin activity  
- Real-world framing:
  - “Admins don’t walk around with master keys all day”
  - They check out the key only when they need it  
  - PIM controls who gets the key, for how long, and records its use
